# コードレビュー

`{{target}}` のコードをレビューしてください。

## レビュー観点

以下の観点で問題がないか確認してください:

1. **バグ** — ロジックミス、境界値の未処理、null/undefined の未チェック
2. **型安全性** — any の使用、型アサーションの濫用、型の不整合
3. **エラーハンドリング** — エラーの握りつぶし、不適切な fallback
4. **パフォーマンス** — 不要なループ、N+1 問題、メモリリーク
5. **可読性** — 不明瞭な命名、複雑すぎる条件分岐、マジックナンバー
6. **セキュリティ** — インジェクション、秘密情報のハードコード

{{#if focus}}
### 重点観点

特に **{{focus}}** に注目してレビューしてください。
{{/if}}

## 出力フォーマット

レビュー結果は以下の形式で出力してください:

### 🔴 問題（修正必須）

- **[ファイル:行]** 説明

### 🟡 警告（修正推奨）

- **[ファイル:行]** 説明

### 💡 提案（任意）

- **[ファイル:行]** 改善案

### 📝 総評

全体的な品質評価と改善の方向性を 2〜3 文で記述してください。
